# 科技社团云展教师上传与维护数据操作手册（中文）

本文面向参与“普陀区科技社团互动云展”的社团教师，目标是让教师在不改代码的前提下，独立完成数据上传、修改、检查与常见问题处理。

## 一、你要维护什么

云展内容由 3 张 CSV 表驱动：

1. `club_profile`（社团表）
作用：展示每个社团的基本信息、简介与社团层面成果。

2. `student_artifact`（学员成果表）
作用：展示“全员成果墙”的每一张成果卡。

3. `media_asset`（素材表）
作用：给社团或成果卡绑定图片/视频素材链接。

一句话理解关系：
- 社团表定义“社团是谁”；
- 成果表定义“学生做了什么”；
- 素材表定义“展示什么图/视频”。

## 二、日常维护总流程（推荐按这个顺序）

1. 打开在线表格（社团表、成果表、素材表）。
2. 先改文字信息（社团简介、成果说明等）。
3. 再上传素材到统一云盘，复制公开链接填到素材表。
4. 检查关键字段（ID、类型、链接）是否正确。
5. 打开云展页面，点击“刷新最新数据”。
6. 抽查展示效果（首页统计、社团卡、成果墙、详情弹窗）。

## 三、详细操作步骤

### 步骤 1：新增一个社团（如需）

在 `club_profile` 新增 1 行，至少填写：
- `club_id`（唯一，如 `C010`）
- `club_name`
- `teacher`
- `status`（填 `active`）

建议同时填写：`grade_range`、`club_category`、`intro`、`cover_url`。

注意：
- `club_id` 一旦使用，不要随意改。
- 如果社团暂不展示，`status` 改为 `archived`。

### 步骤 2：新增一条学员成果卡

在 `student_artifact` 新增 1 行，至少填写：
- `artifact_id`（唯一，如 `A108`）
- `student_alias`（化名，不填真实姓名）
- `grade`
- `club_id`（必须是社团表中已有的 ID）
- `artifact_name`
- `artifact_type`（只能填：`作品` / `任务` / `探究` / `表达`）

建议同步填写：
- `keywords`
- `participation`
- `artifact_intro`
- `one_line_harvest`（建议 140 字以内）
- `growth_evidence`
- `teacher_comment`
- `updated_at`

### 步骤 3：上传图片或视频并绑定

优先方式（推荐）：使用系统内置“本地上传并自动填链接”功能。  
备选方式：在云盘上传素材后，复制可公开访问链接，填到 `media_asset`。

当系统已启用统一上传服务时，教师只需：
1. 选择本地文件（图片/视频）。
2. 点击“上传并填入链接”。
3. 系统自动回填 `cover_url` 或 `media_url`。

若手动填写，`media_asset` 字段如下：
- `media_id`：唯一 ID（如 `M530`）
- `owner_type`：`artifact`（给成果卡）或 `club`（给社团）
- `owner_id`：对应 `artifact_id` 或 `club_id`
- `media_type`：`image` 或 `video`
- `url`：公开访问链接（必须 `http` 或 `https`）
- `thumbnail_url`：视频建议填封面图

说明：
- 同一个成果可以绑定多条素材。
- 视频无缩略图时，前台会显示默认占位图。

### 步骤 4：刷新并核验页面

打开云展页面后点击“刷新最新数据”，重点核验：

1. 首页统计是否变化（社团数、学员数、成果卡数、图视频数）。
2. 新增社团是否出现在“社团展区”。
3. 新增成果是否出现在“全员成果墙”。
4. 点击“查看详情”是否能看到完整文字与素材。

## 五、填写规范（请务必遵守）

1. 不要改表头字段名，不要删表头。
2. 一行只填一个对象，不合并单元格。
3. ID 不重复，建议规则：
- 社团：`C` + 3 位数字（如 `C013`）
- 成果：`A` + 3 位数字（如 `A128`）
- 素材：`M` + 3 位数字（如 `M563`）
4. 学员姓名一律使用化名。
5. 链接必须是外部可访问的 `http/https`。
6. `artifact_type` 仅允许 4 种固定值。

## 六、常见问题与处理

### 1）页面提示“发现数据问题”

优先检查：
- 必填字段是否为空。
- `club_id` / `owner_id` 是否引用了不存在的 ID。
- `artifact_type` 是否写成了非标准词（如“项目”“实验”）。

### 2）成果卡没显示

按顺序排查：
1. `student_artifact` 是否新增成功。
2. `club_id` 是否存在且社团 `status=active`。
3. 是否被筛选条件过滤（社团/类型/关键词）。

### 3）图片不显示或视频打不开

- 检查 `url` 是否有效、是否公开可访问。
- 视频建议补 `thumbnail_url`，提高展示稳定性。

### 4）误改了很多数据

联系科技组使用快照回滚：
- 备份：`node scripts/backup_snapshot.mjs`
- 回滚：`node scripts/restore_snapshot.mjs backup/snapshots/<timestamp>`

## 七、建议工作分工

1. 社团教师：负责本社团文字与成果卡录入。
2. 素材管理员：统一素材命名与云盘目录维护。
3. 科技组抽检：每周抽检 3 个社团 + 10 条成果卡。

- [ ] 新增/修改数据后已点击“刷新最新数据”。
- [ ] 所有新增成果均能在成果墙检索到。
- [ ] 随机点开 5 条成果，详情页文字正常、素材可访问。
- [ ] 学员信息均为化名，无真实姓名泄露。
- [ ] 成果类型只使用“作品/任务/探究/表达”。
